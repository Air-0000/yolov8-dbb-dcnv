# 血细胞自动识别系统

## 一、项目概述

### 1.1 为什么要做这个项目

医院里检查血常规的时候，医生要在显微镜下一张一张地数红细胞、白细胞和血小板。这样做有几个问题：
- **太慢了**：数一张片子要5到10分钟，病人多的时候根本来不及
- **每个人数的不一样**：不同医生数的结果可能不一样，有的人容易漏
- **看久了会累**：时间长了眼睛花，小东西（比如血小板）很容易漏掉
- **不够智能**：以前的老方法拍出来的照片不清楚就不好使了

所以我们想做一个程序，能自动识别血细胞，又快又准。

### 1.2 我们要做到什么

| 指标 | 目标 | 怎么检验 |
|------|------|---------|
| 检测准确率(mAP) | ≥80% | 用标准方法算 |
| 精确率 | ≥75% | 算猜对的里面有多少真对了 |
| 召回率 | ≥85% | 算真有的里面找出来了多少 |
| 一张图处理速度 | ≤10毫秒 | 批量跑平均时间 |

### 1.3 谁负责什么

| 成员 | 负责的事 |
|------|---------|
| 成员A | 设计方案、选模型、写文档 |
| 成员B | 写代码训练模型、调参数、管理代码 |
| 成员C | 处理数据、搭界面、写报告、录视频 |

## 二、总体方案

### 2.1 整个流程是什么样的

```
数据准备 → 训练模型 → 测试模型 → 出结果图片
  ↓           ↓          ↓          ↓
图片+标签    AI学习    看准不准   画框标出来
```

具体来说：
1. **数据部分**：找公开的血细胞图片，告诉程序每张图里细胞在哪里、是什么种类
2. **模型部分**：用深度学习模型训练，让AI学会认细胞
3. **推理部分**：拿训练好的模型识别新图片，画出框和标签
4. **结果部分**：出效果图，算准确率

### 2.2 用了什么技术

| 技术 | 为什么用它 |
|------|-----------|
| Python | 搞AI最常用的语言，什么库都有 |
| PyTorch | 深度学习的框架，好调试，NVIDIA显卡跑得快 |
| YOLOv8 | 目前最好的目标检测工具之一，又快又准 |
| DBB | 一种改进的卷积方法，训练时学得更好，跑的时候不慢 |
| DCNV4 | 另一种改进的卷积方法，能适应细胞的各种形状 |
| CUDA | 让NVIDIA显卡加速计算 |

### 2.3 分了几个模块

| 模块 | 干什么的 | 用的技术 |
|------|---------|---------|
| 数据模块 | 加载图片、做增强、缓存 | YOLODataset |
| 模型模块 | 搭建神经网络 | PyTorch + Ultralytics |
| 训练模块 | 训练模型 | PyTorch + AdamW优化器 |
| 评估模块 | 测模型准不准 | YOLO官方评估工具 |
| 推理模块 | 用模型认新图片 | YOLO.predict() |

## 三、具体怎么做

### 3.1 数据部分

**数据集来源：** BCCD公开数据集，网上能下到，全是血细胞的显微照片。

**数据怎么处理：**
- 所有图片统一缩放到 640×640 大小
- 像素值从0-255归一化到0-1之间
- 做一些随机增强（模糊、变灰、调亮度），让模型更鲁棒

**数据怎么分：**

| 用途 | 图片数 | 标注数 | 比例 |
|------|--------|--------|------|
| 训练用 | 184张 | 2,518个 | 85% |
| 验证用 | 32张 | 459个 | 15% |

**三种细胞的数量：**

| 种类 | 训练集 | 验证集 | 平均每张图 |
|------|--------|--------|-----------|
| 红细胞(RBC) | 2,170 | 399 | 约13个 |
| 白细胞(WBC) | 174 | 32 | 约1个 |
| 血小板(Platelets) | 174 | 28 | 约1个 |

### 3.2 模型部分

#### 3.2.1 这个模型是怎么工作的

YOLOv8 是目前很流行的目标检测模型，它看一眼图片就能知道：
1. 图里有没有细胞
2. 细胞在哪个位置（画个框）
3. 是什么种类的细胞

我们做了两处改进：
- **DBB**：在模型的前半部分用了多分支卷积，训练的时候多个分支一起学，学到更多特征，但用的时候合并成一个，速度不变
- **DCNV4**：在模型的后半部分用了可变形卷积，能根据细胞形状调整感受野，对圆形细胞更友好

#### 3.2.2 训练参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 一次处理几张图 | 8张 | 充分利用显存 |
| 训练几轮 | 50轮 | 看到不再下降了就停 |
| 输入图片大小 | 640×640 | 统一尺寸 |
| 优化器 | AdamW | 最常用的优化器之一 |
| 学习率 | 0.001 | 控制学习速度 |
| 权重衰减 | 0.0005 | 防止过拟合 |
| 学习率预热 | 前3轮 | 刚开始慢慢学 |
| 混合精度 | 关掉 | DCNV4需要关掉才能稳定 |

#### 3.2.3 核心训练代码

```python
# 训练的核心循环
for epoch in range(50):          # 一共训练50轮
    for batch in dataloader:     # 每次拿一批图片
        # 把图片搬到显卡上
        batch['img'] = batch['img'].float().to('cuda') / 255.0
        # 把标签也搬到显卡上
        batch['cls'] = batch['cls'].to('cuda')
        batch['bboxes'] = batch['bboxes'].to('cuda')
        
        # 清零梯度 → 前向传播 → 算损失 → 反向传播 → 更新参数
        optimizer.zero_grad()
        pred = model(batch['img'])
        loss = model.loss(batch, pred)[0].sum()
        loss.backward()
        optimizer.step()
```

#### 3.2.4 训练过程

从第1轮到第50轮，损失值一路下降：

```
第1轮: 损失=41.2
第10轮: 损失=35.1
第20轮: 损失=34.5
第30轮: 损失=33.5
第40轮: 损失=32.7
第50轮: 损失=30.5
```

损失一直在降，说明模型在持续学习。

#### 3.2.5 最终成绩

| 类别 | 精确率 | 召回率 | mAP@0.5 | mAP@0.5:0.95 |
|------|:-----:|:------:|:-------:|:------------:|
| **整体平均** | **81.9%** | **90.5%** | **84.0%** | **51.3%** |
| 红细胞 | 79.0% | 75.2% | 76.0% | 49.7% |
| 白细胞 | **97.0%** | **100%** | **97.3%** | 68.3% |
| 血小板 | 69.8% | 96.4% | 78.6% | 35.9% |

**所有指标都达到了目标：**
- mAP@0.5 = 84.0% > 目标80% ✅
- 召回率 = 90.5% > 目标85% ✅
- 精确率 = 81.9% > 目标75% ✅
- 一张图处理 = 2.6毫秒 < 目标10毫秒 ✅

**解决了什么问题：**
1. 机器自动数细胞，不用人工显微镜看了
2. 每次数得都一样，没有人为主观差异
3. 小目标（血小板）也能检出96%

### 5.2 遇到过什么坑

| 问题 | 怎么解决的 |
|------|-----------|
| 新显卡跑训练就崩 | 不用YOLO自带的训练，改成手动写训练循环 |
| 模型分类学不会 | 发现有个参数传错了，导致标签全被过滤掉，去掉就好了 |
| 配置文件类别数不对 | 配置文件写了80类，数据只有3类，改过来 |
| 代码推送GitHub太慢 | 加了个.gitignore，不传没用的文件 |

### 5.3 还有什么可以改进

| 不足 | 以后怎么改 |
|------|-----------|
| 血小板不太准(35.9%) | 用更高清的图片，或者多尺度训练 |
| 损失降到30就下不去了 | 换更大更好的模型 |
| 没有AI对话功能 | 接个大模型，让AI解读结果 |
| 没有网页界面 | 用Streamlit或Gradio搭一个 |
| 数据只有184张 | 加更多数据增强，或者找更多图片 |
